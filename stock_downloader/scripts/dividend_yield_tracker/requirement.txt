需求文档：A 股红利再投资与防御工具 (dividend_yield_tracker.py)1. 任务目标构建一个专门处理“分红数据”的同步工具。通过抓取 Tushare 的分红送股信息，计算股票的静态与动态股息率，支撑 Agent 构建“杠铃策略”中的高股息底仓（即 5% 以上年化收益的“现金牛”）。2. 核心逻辑：杠铃策略的“防御端”在长线思维中，红利不仅是收益，更是安全垫。Agent 目标： 寻找“分红稳定（连续 5 年以上）”、“派息率合理（不掏空公司）”、“股息率覆盖无风险利率（通常 > 3-5%）”的公司。关键概念： * 股权登记日 (Record Date)： 必须在此日期收盘前持有股票才有分红权。除权除息日 (Ex-date)： 股价在此日会进行扣掉分红的处理。分红周期： A 股通常在年报后（5-7 月）集中分红。3. 详细功能需求A. 数据抓取范围 (Tushare 接口)分红送股 (dividend)： 核心接口。关键字段： ann_date (公告日), end_date (分红年度), cash_div_tax (每股派息/含税), record_date (登记日), ex_date (除权除息日), div_proc (实施进度：如果是“预案”则仅参考，如果是“实施”则是确定性数据)。B. 计算指标需求 (脚本内处理)脚本不仅是下载，还需要为 Agent 预计算以下指标：静态股息率 (Static Yield)： $当年派息 / 当前股价$。派息率 (Payout Ratio)： $总分红金额 / 当年净利润$（需关联 income 表数据）。分红连续性： 该公司过去 5 年是否每年都分红？分红金额是否在增长？C. 同步策略全量回溯： 至少下载过去 10 年的全市场分红记录。高频监控： 在 4 月至 6 月（年报披露季），脚本需每日运行，抓取最新的分红预案公告。4. 存储架构建议 (针对你的 MongoDB 环境)推荐集合名：fact_dividend_history字段名说明Agent 筛选逻辑ts_code股票代码-end_date分红所属年度db.count({ts_code: "xxx", end_date: {$gt: "2020"}})cash_div_tax每股派息 (元)用于计算总收益ex_date除权除息日提醒 Agent 在此日期前买入div_proc实施进度仅关注“实施”或“公告”状态payout_ratio派息率 (计算得出)过滤掉派息率 > 100% 的不可持续公司5. Agent 专属 Tool 接口预留开发人员应封装以下 API 供 OpenClaw 调用：get_cash_cow_list(min_yield=0.05)：逻辑： 结合当前最新股价（来自 market_quota_syncer）和最新派息数据，返回股息率大于 5% 的股票清单。check_dividend_sustainability(ts_code)：逻辑： 检查过去 3-5 年分红是否稳定，避免因一次性“特别分红”导致的股息率虚高。get_upcoming_dividends()：逻辑： 找出已经公告但尚未到达“登记日”的股票，供 Agent 进行抢权参考。6. 开发者注意事项（避坑指南）含税与税后： Tushare 提供的是含税派息。对于个人投资者，持有时间不满 1 个月、1 年，税率不同。Agent 逻辑中可以默认按持有一年以上（免税或 5% 税）来简化计算。送转股干扰： 脚本要区分“现金分红”和“送转股”。长线策略优先关注现金分红。股价匹配： 计算动态股息率时，必须确保获取的是最新的收盘价。如果股价近期暴涨，原本 5% 的股息率可能降到 3%，Agent 需及时感知。
