需求文档：A 股宏观政策与叙事抓取工具 (macro_policy_scrapper.py)1. 任务目标构建一个自动化“情报收集站”。通过聚合 Tushare 官方新闻流、政府官网公告及《新闻联播》文本摘要，提取关键词和政策导向。该工具旨在支撑 Agent 的**“北极星思维”**：识别国家战略投资方向（如 2026 年关注的“人工智能深度融合”、“低空经济”或“内需振兴”），从而锁定长期赛道。2. 核心逻辑：定性分析转化为定量信号政策思维在 A 股不是靠“猜”，而是靠“高频关键词”的出现：信号增强： 某个词（如“新质生产力”）在重要会议中出现的频率从 1 增加到 10，代表大级别行情的起点。政策周期： 识别当前处于“政策酝酿期”、“密集发布期”还是“效果观察期”。Agent 目标： 避免在政策冷淡期盲目进攻，在政策风口期敢于重仓。3. 详细功能需求A. 数据抓取范围Tushare 接口需求：新闻快讯 (news)： 实时抓取主流财经媒体（如 7x24 快讯）。经济日历 (eco_cal)： 监控重要数据发布日（如 CPI, PPI, PMI, 降准降息日）。网页爬虫/API 集成（建议）：《新闻联播》文字版： 抓取每日文字摘要，提取前 5 条新闻的关键词。重要会议纪要： 包含但不限于：中央政治局会议（每季度）、中央经济工作会议（12 月）、全国两会（3 月）。B. 数据处理逻辑 (核心竞争力)关键词提取： 脚本需接入简单的 NLP 库（或直接调用 LLM 接口），对新闻进行摘要，提取“行业/板块”关键词。情感极性分析： 判断政策基调是“支持、稳健、规范、严打”中的哪一类。事件打标： 将新闻与具体的 asset_registry 中的板块进行关联。例如：新闻提到“算力布局”，脚本应自动打上“通信/计算机”标签。4. 存储架构建议 (针对你的 MongoDB 环境)推荐集合名：fact_macro_narratives字段名说明Agent 逻辑应用event_date事件/新闻发生日期匹配行情波动时间点source来源 (新闻联播/Tushare/政府网)权重判断 (官方来源权重最高)title标题/会议名称-keywords核心提取词 (数组)db.find({keywords: "低空经济"})summaryLLM 生成的 100 字摘要供 Agent 阅读理解上下文importance重要程度评分 (1-5)过滤琐碎杂音5. Agent 专属 Tool 接口预留开发人员应封装以下接口供 OpenClaw Agent 调用：get_current_policy_focus()：逻辑： 统计过去 30 天出现频率最高的前 5 个行业关键词。输出： “当前政策聚焦：1. AI 基础设施, 2. 半导体自主, 3. 养老金融...”check_event_impact(sector)：逻辑： 检索特定行业（如“医药”）近期是否有重大利空或利好政策。is_macro_safe_period()：逻辑： 综合评估是否有系统性风险（如加息预期、严监管信号）。6. 开发者注意事项（避坑指南）噪音过滤： 财经新闻非常多，脚本必须过滤掉“xx 股票涨停”、“xx 机构预测”等垃圾信息，只保留官方、权威、宏观的信息。LLM 成本控制： 如果调用 GPT/Claude 进行摘要，建议先进行本地关键词过滤，只针对高价值新闻进行处理，节省 API 成本。时间敏感性： 政策信号通常在收盘后发布，脚本需在 19:00 - 21:00 之间完成主要数据的更新，以便 Agent 在次日开盘前给出建议。
